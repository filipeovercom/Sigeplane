Como Administrador ou Coordenador desejo cadastrar Recursos para serem utilizados como sugestão no preenchimento do plano de ensino pelo professor.<br/><br/>Para que o aluno obtenha uma habilidade é nessário que o professor ministre os conteúdos adequados e da forma adequada. Para que isso aconteça é necessário a utilização de alguns recursos.<br/><br/>São exemplos de recursos:<br/><ul>
<li>Computador</li><li>Software Excel</li><li>Datashow</li><li>Quadro Branco</li></ul><p/>

Requirement Type Status Difficulty Priority Stability
Alteração de Recursos Functional Proposed Medium Medium Moderate
Details:
A tela de listagem de recursos deve possuir a opção para alteração de um recurso.<br/><br/>Ao acionar a ação o sistema deve exibir um formulário para a alteração das informações:<br/><ul>
<li>Nome do Recurso</li><li>Descrição sobre o recurso</li></ul><p/>
Cadastro de novos recursos Functional Proposed Medium Medium Moderate
Details:
A tela de listagem de recursos deve possuir a opção para cadastrar novos conteúdos.<br/><br/>Ao acionar a ação o sistema deve exibir um formulário com os seguintes campos:<br/><ul>
<li>Nome do Recurso</li><li>Descrição sobre o Recurso</li></ul><p/>
Exclusão de Recursos Functional Proposed Medium Medium Moderate
Details:
A tela de listagem de recursos deve possuir uma opção para exclusão de recursos.<br/><br/>Ao acionar a ação o sitema deve exigir a confirmação por parte do usuário. Caso o usuário confirme, o sistema deve eliminar o recurso da base de dados.<br/>
Listagem de Recursos Functional Proposed Medium Medium Moderate
Details:
O sistema deve possuir uma tela para listagem de recursos cadastrados no sistema.<br/><br/>Nesta tela o usuário deve possuir acesso as opções de cadastro, alteração e exclusão de conteúdos. Para exibição dos recursos cadastrados o sistema deve exibir uma tabela com as seguintes colunas:<br/><ul>
<li>Nome</li><li>Descrição</li><li>Dados de Utilização</li></ul><p/>
Constraint Type Status
Controle de Acesso Pre-condition Approved
Details:
Para acessar esta funcionalidade o usuário deve possuir o perfil Administrador e/ou Coordenador.<br/>
Validação de Cadastro/Alteração Post-condition Approved
Details:
Para cadastro e alteração de um recurso é obrigatório a informação do nome.<br/>
Scenario Type
Listagem de Recursos Basic Path
Details:
Steps:
1. Aciona o menu Recursos na seção Gerenciamento de Cadastros<br/>
2. Redireciona o usuário para a tela de listagem<br/>
3. Verifica as permissões de acesso<br/>
4. Exibe os recursos cadastrados na base de dados<br/>
Exceptions:
4a. Cadastro de Novo Recurso (at End)
4b. Alteração de Recurso (at End)
4c. Exclusão de Recurso (at End)
Cadastro de Novo Recurso Alternate
Details:
Steps:
1. Aciona ação Cadastrar novo Recurso<br/>
2. Exibe formulário para preenchimento das informações<br/>
3. Preenche as informações e clica em salvar<br/>
4. Valida as informações e salva o registro na base de dados<br/>
Alteração de Recurso Alternate
Details:
Steps:
1. Aciona a ação Alterar Recurso<br/>
2. Exibe formulário para alteração das informações<br/>
3. Altera as informações desejadas<br/>
4. Valida as informações e registra a alteração na base de dados<br/>
Exclusão de Recurso Alternate
Details:
Steps:
1. Aciona ação Excluir Recurso<br/>
2. Exibe tela de confirmação da ação<br/>
3. Confirma a ação<br/>
4. Elimina o recurso da base de dados<br/>
